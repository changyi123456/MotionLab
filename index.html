Core Math

技術運算原理

2D Tracker：從影像像素到物理量

2D 模式以單一影片為基礎。使用者先設定原點、比例尺與座標軸方向， 程式再把每一幀標記到的像素位置轉成真實世界座標。若影像水平與實驗座標有夾角， 會以座標旋轉校正；若已知距離為 L、對應像素距離為 p，比例可寫成 p / L。

軌跡資料可進一步產生位置、速度、加速度、路徑長、角度與圖表資料。 自動追蹤則以使用者給定的初始目標區域為模板，在後續幀搜尋最相近位置； centroid snap 可協助把點吸附到亮點或暗點中心，降低人工點選誤差。

2D 適用場景

  • 拋體運動、簡諧運動、碰撞、圓周運動等平面運動。
  • 攝影機固定、量測平面明確，或物體主要在單一平面內移動。
  • 需要快速得到 t、x、y、vx、vy、ax、ay 等資料表。

3D Tracker：雙視角 XZ / YZ 融合

目前 MotionLab 3D Workbench 採用兩個近似正交視角：相機 A 量到 XZ 平面， 相機 B 量到 YZ 平面。兩邊各自完成原點與比例尺校正後，程式依同步幀與 FPS 將 兩條時間序列對齊，取共同時間範圍，融合出 X、Y、Z 軌跡。

為了處理兩台相機校正後仍可能存在的 Z 軸尺度差與近距離視差，3D 引擎使用 Z 軸一致性做自校正：同一個物體在兩個視角中應該有相同的 Z。程式會估計 Z 對齊的 scale、offset，以及一階視差修正參數，並回報共同有效幀數與 Z 殘差 RMS。

3D 適用場景

  • 需要 X、Y、Z 三維軌跡，但希望用較簡單的雙視角設定完成。
  • 兩台攝影機可分別拍到 XZ 與 YZ 分量，且時間能夠同步。
  • 想快速比較 3D 軌跡圖、資料表與輸出 CSV/Python 繪圖腳本。

How To Use

操作說明

2D Workbench 操作流程

  1. 開啟影片，確認 FPS、總幀數與畫面比例。
  2. 使用比例尺與座標工具，設定真實距離、原點與軸向。
  3. 建立 Point Mass 或其他分析物件，逐幀標記位置。
  4. 需要精細標記時可使用影片放大縮小與逐幀快捷鍵。
  5. 可啟用 centroid snap 或 auto track，加快連續幀追蹤。
  6. 到資料表與圖表檢查位置、速度、加速度，再匯出 CSV。

3D Workbench 操作流程

  1. 載入相機 A：XZ 視角影片；載入相機 B：YZ 視角影片。
  2. 兩個視角都設定原點、比例尺與同步幀。
  3. 在兩個視角追蹤同一物體，保持標記點代表同一物理位置。
  4. 執行 Z 融合，檢查共同有效幀數、Z 殘差 RMS 與融合摘要。
  5. 切換到 3D 軌跡圖，使用頂視、正視、側視與等角視角檢查軌跡。
  6. 匯出資料包，取得 CSV、3D 圖、四視圖、Python 腳本與專案檔。
